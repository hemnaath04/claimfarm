# ClaimFarm API

All endpoints are JSON unless otherwise noted. Mutating endpoints set
`Cache-Control: no-store`. Auth is via the `cf_session` HTTP-only cookie
set by `/auth/sign-in` and `/auth/sign-up`.

The full machine-readable schema lives at `/openapi.json` and an
interactive explorer at `/docs` (Swagger UI, generated by FastAPI).

---

## Auth

### `POST /auth/sign-up`
```json
{ "email": "you@org.org", "password": "min8chars", "name": "Jane Doe", "org": "Demo NGO" }
```
**201** `{ "user_id": "usr_...", "session": "sess_...", "verification_email_sent": true }`
- Sets `cf_session` cookie. First user is automatically `role=owner`.
- Triggers a welcome email + email-verification token.

### `POST /auth/sign-in`
```json
{ "email": "...", "password": "..." }
```
**200** `{ "user_id": "...", "session": "..." }`
**401** if credentials wrong or account disabled.

### `POST /auth/sign-out`
Clears the cookie + revokes the server-side session.

### `POST /auth/reset`
```json
{ "email": "..." }
```
Always **200** to avoid leaking which emails are registered. Issues a
1-hour reset token via email (logged when SMTP is unset).

### `POST /auth/reset/confirm`
```json
{ "token": "...", "password": "newpassword" }
```

### `GET /auth/verify?token=...`
Marks the user's email as verified.

### `GET /auth/me`
**200** current user (role, identity_status, etc.).

## Claims (adjuster + insurer)

### `GET /api/claims?status={pending_review|approved|rejected|all}`
```json
{
  "items": [
    {"claim_id": "CF-...", "status": "pending_review", "farmer_name": "...", "crop_type": "...", "estimated_loss_usd": 1234}
  ],
  "stats": { "total": 4, "pending_review": 3, "approved": 1, "rejected": 0, "submitted": 0, "paid": 0 }
}
```

### `GET /api/claims/{claim_id}`
Returns the full `Claim` (damage, weather, corroboration, forensics) +
similar past claims (DashVector hits) + fraud flags + pdf_path.

### `POST /api/claims/{claim_id}/decision`
```json
{ "decision": "approve" | "reject" | "request_info", "notes": "..." }
```
- `approve` → submits to mock insurer + records the terminal status.
- `reject` → final.
- `request_info` → goes back to `pending_review` with a follow-up note.

### `GET /api/claims/{claim_id}/localized_reply`
On-demand localized farmer message for the claim's current status.

## Billing

### `POST /api/billing/checkout?plan=growth`
Returns a hosted-checkout session from the configured payments provider
(Paddle / LemonSqueezy / Razorpay), or a stub when `PAYMENTS_PROVIDER=none`.

### `POST /api/billing/webhook`
Provider-agnostic webhook receiver. Reads the `Paddle-Signature`,
`X-Signature`, or `x-razorpay-signature` header (whichever the
configured provider sets) and verifies HMAC against the matching
`*_WEBHOOK_SECRET`.

## Channels (webhook receivers)

### `POST /twilio/inbound`
Form-encoded Twilio webhook. Returns TwiML ack; pipeline runs in a
background task.

### `POST /bird/inbound`
Bird Conversations API JSON payload. Logs raw to
`/tmp/claimfarm_bird_payloads.jsonl` for schema verification.

### `POST /telegram/inbound`
Telegram Bot API Update payload. Handles `/start`, location-share,
photo (compressed) and document (uncompressed image) message kinds.

## Rate limits

- 60 requests/IP/minute on `/api/*` and `/auth/*` (in-process middleware)
- 429 response with `Retry-After: 60` when exceeded

## Error format

All errors follow FastAPI's default:

```json
{ "detail": "..." }
```

with the appropriate HTTP status code.
